
- DISTRIBUZIONE: se link a pagina e non a file:
    - compilare SOLO “URL di accesso” e non “URL di download”
- DISTRIBUZIONE: non mettere link a descrizione formato per formati standard (es: CSV)
- DISTRIBUZIONE: titolo nel formato "nome dataset in formato "tipo di formato”
- Per alcuni dataset l'unica risorsa disponibile è una URL ad una banca dati. Non va bene. L'url per il Download deve essere un file. Vi inviterei pertanto a rimuovere tali dataset se non fosse possibile inserire una url diretta per il download.
- La proprietà obbligatoria per le distribuzioni è `dcat:accessURL`definita come
	“Inserire l’URL di una pagina web tramite cui si possa accedere alla distribuzione. Essa può essere anche una pagina informativa che fornisce le indicazioni su come ottenere il dataset oppure l’URL di endpoint di accesso al dataset (e.g., SPARQL endpoint). Si consiglia di indicare l’indirizzo completo, comprensivo anche di protocollo (es. [http://](http:)). **Si raccomanda di usare questa proprietà, e NON la proprietà** [URL di download](https://docs.italia.it/italia/daf/linee-guida-cataloghi-dati-dcat-ap-it/it/stabile/distribuzione_elementi_opzionali.htmlurl-di-download-dcat-downloadurl), **quando non è assolutamente una pagina di download o non si è sicuri che lo sia.”**
